Description: 
kvserver.java starts the server using TThreadPoolServer.

Log.java is a data structure for storing the start time, end time, operation etc for each operation.

Graph.java is a data structure which creates a graph for the given system and provides methods for adding edges and checking for cycles.

kvclient.java takes server's IP address and port number as argument and spawns multiple threads (using ExecutorService) which randomly perform get and set operations on the KV Store. After all the threads are done, it performs analysis by generating a graph according to the HP consistency paper. We are enabling concurrent requests by using multiple threads. 

KVStoreHandler.java handles the commands and gives appropriate Results. It maintains a hashmap to store the data.

Running commands:
Go to the folder 'Executables' where kvserver.jar and kvclient.jar are located.

To run our server (good server):
java -jar kvserver.jar 

To run the client:
java -jar kvclient.jar -server localhost:9090

Note 1: Execution of the client takes around 30 seconds (We are making about 500 requests to the server)
On Windows, check the error code of the last execution by running "echo %errorlevel%
On Linux, the command is echo $?

-----------------------------------------------------------------------------------------
Using the Code:(Use Eclipse)
Create empty Java project in Eclipse.
Right click the empty project and select import
In pop-up box: General -> FileSystem
Navigate to location of KVStore folder and Import it.

-----------------------------------------------------------------------------------------
For Compilation: 
File -> Export Select runnable JAR file.
launch configuration = kvserver - KVStore (for server - kvserver.jar)
launch configuration = kvclient - KVStore (for client - kvclient.jar)
Specify the location and name as kvserver.jar and kvclient.jar respectively.
Click Finish.
